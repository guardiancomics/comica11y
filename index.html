ComicA11y is an experiment that aims to achieve an all-inclusive online comic reading experience.

Because comics should be universal.

What you're about to read may look and feel like a regular online comic but take a closer look and explore the different modes, built in to aid the reading experience and provide options for those who may need them.

What ComicA11y does

Focus management

For users that rely on navigating using the keyboard, the tab key toggles between frames. Additionally the focus state for each frame contains a movement based thickening of the frame border.

Engineered for screen readers

The focus state for each frame is a figcaption specifically written for screen readers. It describes the setting, characters speaking along with the dialogue along with anthing necessary to convey the material.

Best practices

The comic includes HTML5 elements to provide a more semantic markup along with ARIA attributes to aid assitive technology.

High contrast mode

Each frame contains the source for full colour and simple lineart. For the partialially blind, a high contrast image is far more suited. With this setting engaged the asset is switched to the high contrast version.

Colour blindness filters

Using SVG filters we're able to alter the hue and saturation of the image to aid in eliminating or altering the problematic spectrums of colour for each of the types of colour blindness.

Closed captions

If there's too much visual stimulation within the comic frame, closed captions provides a script version of each frame. We include the name of each person speaking along with an avatar so the reader knows which character the text belongs to.

Increase / Decrease font-size

While in closed caption mode, you can control how large or small you prefer the font.

Selectable text

Each speech bubble is overlayed on top of the artwork using font-face and custom comic book fonts.

Translatable text

Because the text is rendered to the page, it means we can translate it, you're welcome! Or should I say... gracias!

Programatic bubbles

Using javacript, svg and a lot of math each speech bubble is constructed on the fly based around the width and height of the text contained within. Each bubble accepts paramaters which set the position of the spike (left, center or right) the direction the spike curls (left, center or right) and how long the spike is (short, normal, long, variable). Because it's constructed on the fly, it can be moved, altered or resized.

Responsive layout

Using CSS grid and media queries we're able to adapt the layout of the comic to suit any screen size.

Responsive images

Using the picture element and source queries, we're able to serve the correct size and density to suit the device.

Lazyloading

Each image is fetched from the server only once the page has loaded ensuring performance comes first.

RTL / LTR mode

Because not every langauge reads left to right like Western Culture, there is the option to inverse the layout so the order of the frames reverses along with all of the bubbles and artwork.

The problem

Here's a typical example of how an online comic author might present their comic strip.

  • Little to no information provided in the alt text or title.

  • A high resolution image serving all screen and device sizes. This is often a high resolution asset ranging from 900 - ~3500px wide and file size anywhere between 100kb - 2.5mb.

  • No keyboard ARIA information for assistive technologies.

  • No colour options for people who struggle with colour blindness or vision impairments.

  • No closed captions for assistive technology to convey the speech bubbles to text.

  • No right to left reading mode.

  • No way to translate text.

The objectives

Must have features

  • Colour blind and high contrast mode.
  • Closed captions.
  • Translatable.
  • RTL/LTR.
  • Resizable text.
  • Responsive / fluid layout.
  • ARIA best practices.

Nice to have features

  • Toggle closed captions on with browser zoom and system zoom.
  • Autoplay mode.
  • Audio mode.
  • Responsive image sizes.
  • Keyboard commands.
